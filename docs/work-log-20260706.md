 ## 三、项目约定
 
 ### 修改代码的规则
 
 1. 修改任何文件前不擅自运行修复脚本（`fix_*.py`、`_fix_*.py` 等）
 2. 不自动执行 `rstrip()` 或其他字符串操作来修 JS
 3. JS 文件需要验证时手动跑 `node --check 文件名`，不自动批量执行
 4. 遇到 MySQL 连接、nginx 重启、PHP 配置修改等高危操作，必须先告知用户
 5. 改坏了用 git checkout 恢复，不要手动改来改去
