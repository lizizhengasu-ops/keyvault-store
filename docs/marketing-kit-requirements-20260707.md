# Marketing-Kit — 需求定义与方案框架

## 一、背景

我们正在搭建 KeyStarter 独立站的完整运营体系。前端已完成 (Vite + React + TypeScript, 8117端口)，
WooCommerce 后端已连接 (aliali.local, 20 个产品已同步)，SEO 已集成。现在是补齐营销能力。

## 二、当前已有能力 (WooCommerce 内置)

| 能力        | 状态   | 说明                              |
|------------|--------|-----------------------------------|
| 交易邮件    | ✅ 基础 | 下单确认、发货通知等 WooCommerce 自带 |
| 优惠券      | ✅ 完整 | WooCommerce 优惠券系统              |
| 基础报表    | ✅ 基础 | WooCommerce 自带销售报表             |
| 弃购挽回    | ❌ 无   | 需要购物车放弃自动提醒               |
| 营销邮件    | ❌ 无   | 没有邮件营销能力                     |
| 客户分群    | ❌ 无   | 不能按购买行为分群                   |
| 转化跟踪    | ❌ 无   | 没有邮件 -> 购买的转化归因           |

## 三、功能需求

### 3.1 交易邮件增强 (P0 - 必须)

WooCommerce 自带交易邮件只覆盖基础场景。需要增强:
- 订单确认邮件: 包含产品激活码/序列号 (软件授权产品的核心交付)
- 发货通知: 包含下载链接和激活说明
- 以 KeyStarter 品牌模板发送 (非默认 WooCommerce 样式)

### 3.2 弃购挽回 (P0 - 必须)

独立站最大收入损失来自购物车放弃。需要:
- 检测未完成结账的购物车
- 发送提醒邮件 (30分钟后 / 24小时后 / 72小时后三封)
- 可附带限时优惠码提升转化
- WooCommerce 不自带此功能，需定制

### 3.3 营销邮件 (P1 - 重要)

- 新产品/新分类上架通知
- 促销活动通知 (Bundle 折扣、限时优惠)
- 客户欢迎序列 (新注册用户自动发送)
- 节日/季节性活动

### 3.4 邮件模板与品牌 (P1 - 重要)

- 所有邮件使用统一的 KeyStarter 品牌模板
- 支持 HTML 邮件设计
- 响应式 (移动端适配)

### 3.5 分析与报告 (P2 - 一般)

- 邮件送达率、打开率、点击率统计
- 邮件带来的转化收入
- 客户生命周期价值跟踪

### 3.6 受众分群 (P2 - 一般)

- 按购买历史分群 (曾购买 Windows vs Office vs Server)
- 按消费金额分群
- 按注册时间分群
- 按弃购行为分群

## 四、邮件发送方案 (待决策)

| 方案 | 描述 | 优点 | 缺点 |
|------|------|------|------|
| A: Mailchimp/SendGrid | 用 SaaS 发邮件 | 功能全/送达率高/有免费额度 | 依赖外部服务/可能产生月费 |
| B: WordPress SMTP | wp_mail + SMTP 插件 | 免费/无外部依赖 | 功能弱/易进垃圾箱/无模板 |
| C: SendGrid(交易) + Mailchimp(营销) | 分层策略 | 交易邮件可靠+营销功能全 | 管理两个服务/轻度复杂 |

## 五、技术约束

- 前端: Vite + React SPA (不直接发邮件)
- 后端: WordPress + WooCommerce REST API
- 现有集成: seo-kit (CLI 调用)、wordpress-kit (WooCommerce 连接)
- 所有 kit 独立隔离，只能通过 CLI/HTTP 调用
- 品牌: KeyStarter (非 KeyVault)
